# ADR-003: 전투력 공식 설계

## 상태
확정

## 배경
MVP의 목표는 "개인 투자자들의 결속력을 높여 대주주와 같은 효과를 얻는 것"이다.
전투력 공식을 어떻게 설계하느냐에 따라 이 목표를 제대로 반영할 수 있는지가 결정된다.

여러 후보 공식을 검토했다.

### 후보 1 — 원 지표 (기각)
```
power = 참여자수 × 총보유주식수 × 현재가
```
삼성전자 같은 대형주가 무조건 유리하다. 회사 규모가 다른 왕국끼리 공정한 경쟁이 불가능하다.

### 후보 2 — log-log 공식 (기각)
```
power = log(총보유금액 + 1) × log(참여자수 + 1)
```
총보유금액 절대값이 점수를 지배한다. 시총 100억 기업에서 2억 보유한 결속력 높은 왕국이
삼성 왕국보다 낮은 점수를 받는 역전 현상이 발생한다.

### 후보 3 — 참여율 기반 공식 + 승수 (채택)
```
power = (총보유금액 / 시가총액) × log(참여자수 + 1) × 1,000,000
```

## 결정
`power = 참여율 × log(참여자수 + 1) × 1,000,000` 공식을 채택한다.

- `참여율 = (총보유주식수 × 현재가) / 시가총액`
- 승수 `1,000,000`은 DB 저장 가능한 스케일로 변환하기 위한 고정 상수

## 이유

**공정성**: 시총을 분모로 두어 회사 규모와 무관하게 "비율"로 경쟁한다.

```
시총 100조, 1조 보유, 참여자 10명 → 참여율 1%
시총 100억, 1억 보유, 참여자 10명 → 참여율 1%
→ power 동일 (공정)
```

**결속력 방향성**: `log(참여자수 + 1)`로 소수 유저 몰빵보다 많은 참여자가
고르게 보유할수록 유리하게 만든다. log 함수 특성상 초기 참여자 증가 효과는
크고, 대규모 참여자에서는 완만하게 증가한다.

**승수 `1,000,000`의 역할**: 순수 스케일 조정이다. 공식의 비율 관계는 유지된다.
유저가 아무리 늘어도 왕국 간 상대 비율은 변하지 않는다.

## 트레이드오프 및 알려진 한계
- 현재 공식은 "몇 명이 얼마나 들고 있냐"만 보고, 보유량이 고르게 분산됐는지는 보지 않는다.
  (예: 1명이 9,900주 + 99명이 각 1주 vs 100명이 각 100주 → 동일 점수)
- 진짜 결속력을 완전히 반영하려면 분산도(표준편차, HHI 등)가 필요하나
  MVP 복잡도를 고려해 2단계로 이연한다. (ADR-004 참고)
